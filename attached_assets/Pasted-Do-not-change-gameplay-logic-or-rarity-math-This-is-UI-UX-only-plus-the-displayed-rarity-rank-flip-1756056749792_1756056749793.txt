0) Global responsive rules (apply everywhere)
	•	No fixed pixel sizes for layout. Use fluid units: clamp(), min(), max(), %, vmin/vw, and rem.
	•	Typography: use a fluid scale (e.g., clamp(12px, 2.2vmin, 18px) for tile text; clamp(18px, 3.2vmin, 22px) for scoreboard values).
	•	Spacing: use clamp() for gaps, paddings, and border radii (e.g., gap: clamp(10px, 1.8vmin, 16px)).
	•	Images/avatars: max-width: 100%, height: auto. No hard-coded widths.
	•	Overflow: Any scrollable region must hide horizontal overflow and avoid double scrollbars. Use overscroll-behavior: contain for modals.
	•	Breakpoints: Prefer content-driven CSS (grid auto-fit/auto-fill, minmax()) over many media queries. Use the few listed below only where needed.
	•	Safe areas: include viewport-fit=cover and env(safe-area-inset-*) padding on the app shell.

Fit logic
	1.	Try full name (First Last [Suffix]) with 2-line clamp:
display:-webkit-box; -webkit-line-clamp:2; -webkit-box-orient:vertical; overflow:hidden; word-break:break-word; hyphens:auto;
	2.	If still overflows (measure via ref), render F. Last[ Suffix] on one line with ellipsis:
white-space:nowrap; overflow:hidden; text-overflow:ellipsis;
	3.	If F. Last still too long, the last name ends with an ellipsis automatically (browser truncates).

	•	Preserve multi-word/hyphenated surnames and suffixes (Jr., Sr., II, III, IV, V).
	•	Accessibility: title and aria-label must always contain the full, original name.

Responsive acceptance
